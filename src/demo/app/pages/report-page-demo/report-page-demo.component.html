Implementation Tips
  • Data Source: Implement ReportPageDataSource interface - getData() is required, other methods are optional
  • Widgets: Configure widgets with type, size, position, and data properties
  • Charts: Use Chart.js configuration format in widget.config for chart customization
  • Filters: Define filter fields with key, label, type, and options
  • Date Range: Use presets for common ranges (today, lastWeek, lastMonth, lastYear)
  • Auto Refresh: Enable autoRefresh with refreshInterval for real-time dashboards
  • Export: Implement exportReport() in data source for PDF/Excel/CSV export
  • Custom Actions: Add custom toolbar buttons with onClick handlers

Best Practices
  • Widget Layout: Use grid positions and colspan for responsive layouts
  • Loading States: The component handles loading indicators automatically
  • Error Handling: Handle errors in your data source Observable and show appropriate messages
  • Performance: Use refreshWidget() to update individual widgets instead of full refresh
  • Accessibility: The component includes ARIA labels for screen readers
  • Responsive: Widget grid adapts to screen size automatically
  • Fullscreen: Use fullscreen mode for presentations and wall displays
