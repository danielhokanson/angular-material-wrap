Page Stereotypes

Common patterns for transactional applications

These page components represent the 7 most common page stereotypes that cover approximately 80% of functionality in transactional/functional applications:

  • List/Table: Browse hundreds/thousands of records with filtering and sorting
  • Detail/View: Display single records with related data and actions
  • Form/Create-Edit: Data entry with validation and state management
  • Search/Filter: Advanced search with complex filtering capabilities
  • Dashboard/Overview: Metrics, KPIs, and quick actions
  • Workflow/Process: Multi-step operations with approval flows
  • Report/Analytics: Data visualization and reporting

Each page component is highly customizable, supports real-time updates, and includes comprehensive accessibility features.
