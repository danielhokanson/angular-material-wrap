Usage Notes

  • Basic Usage: Use [amwAutoFocus]="true" to automatically focus an element
  • Delayed Focus: Use [amwAutoFocusDelay]="milliseconds" to delay the focus action
  • Conditional Focus: Bind the amwAutoFocus input to a boolean property to control when focus occurs

Behavior Details

Element Selection Priority:

  1. If the host element is focusable (input, button, etc.), it will be focused directly
  2. If the host element is not focusable, the directive searches for the first focusable child element
  3. Focusable children include: input, textarea, select, button, anchor, and elements with tabindex (except tabindex="-1")

Usage Notes

  • Basic Usage: Use [amwCopyToClipboard]="text" to copy specific text to the clipboard
  • Auto-detect: If no text is provided, the directive will attempt to copy the element's text content or value
  • Toast Notifications: Built-in toast messages display on success/error by default (can be disabled)
  • Custom Messages: Use amwCopySuccessMessage and amwCopyErrorMessage to customize toast messages
  • Event Handling: Listen to (amwCopySuccess) and (amwCopyError) for custom handling
  • Disable Toast: Set [amwCopyShowToast]="false" to disable built-in notifications
  • Clipboard API: Uses the modern Clipboard API for secure copy operations
  • Browser Support: Requires HTTPS in production environments (HTTP works on localhost)

Quick Examples

Basic: [amwCopyToClipboard]="text"
Custom message: amwCopySuccessMessage="Copied!"
No toast: [amwCopyShowToast]="false"
With events: (amwCopySuccess)="handleCopy($event)"

Behavior Details

Copy Priority:

  1. If amwCopyToClipboard input has a value, that text will be copied
  2. If the input is empty and the element is an input/textarea, its value will be copied
  3. If the input is empty and the element is not an input, its textContent will be copied

Event Sequence:

  1. User clicks the element with the directive
  2. Directive attempts to copy text using the Clipboard API
  3. On success: amwCopySuccess event is emitted (if subscribed), toast is shown (if enabled)
  4. On error: amwCopyError event is emitted (if subscribed), error toast is shown (if enabled)

Security Considerations:

  • The Clipboard API requires a secure context (HTTPS) in production
  • Localhost is considered secure for development
  • Some browsers may prompt the user for clipboard permission

Usage Notes

  • Basic Usage: Use (amwClickOutside)="handler()" to detect clicks outside an element
  • Event Object: The emitted Event object contains information about the click (target, coordinates, etc.)
  • Event Propagation: Clicks inside the element are stopped from propagating to prevent false triggers
  • Document Listener: The directive attaches a listener to the document to detect all clicks
  • Automatic Cleanup: The document listener is automatically removed when the directive is destroyed
  • Common Use Cases: Closing dropdowns, menus, modals, and dismissing popups
  • Multiple Elements: Can be applied to multiple elements independently on the same page
  • Performance: Uses Renderer2 for efficient event handling and cleanup

Quick Examples

Basic: (amwClickOutside)="close()"
With event: (amwClickOutside)="handler($event)"
Conditional: (amwClickOutside)="isOpen && close()"
Toggle: (amwClickOutside)="isOpen = false"

Behavior Details

How It Works:

  1. When the directive initializes, it attaches a click listener to the document
  2. When a click occurs anywhere on the page, the directive checks if the click target is inside the element
  3. If the click is outside, the amwClickOutside event is emitted
  4. If the click is inside the element, it stops propagation to prevent the document listener from triggering

Event Propagation:

  • Clicks inside the directive's host element have their propagation stopped automatically
  • This prevents the "click outside" event from triggering when clicking inside the element
  • The original click event still triggers normally on the clicked element

Lifecycle:

  • ngOnInit: Document click listener is attached
  • ngOnDestroy: Document click listener is removed to prevent memory leaks
  • The directive handles cleanup automatically - no manual cleanup required

Common Patterns:

  • Dropdown Menus: Close menu when clicking outside the dropdown container
  • Modal Dialogs: Close modal when clicking the backdrop outside the modal content
  • Editable Fields: Save changes when clicking outside an inline edit form
  • Context Menus: Dismiss context menu when clicking elsewhere on the page
  • Popups/Tooltips: Hide popup when user clicks outside the popup area

Directive API Overview

AmwTooltipDirective

Displays custom tooltips with configurable positioning and styling.

  • Selector: [amwTooltip]
  • Inputs: amwTooltip: string, position: 'top' | 'bottom' | 'left' | 'right'
  • Usage: Add to any element to show a tooltip on hover

AmwClickOutsideDirective

Detects and handles clicks outside of an element.

  • Selector: [amwClickOutside]
  • Outputs: (amwClickOutside): EventEmitter<MouseEvent>
  • Usage: Useful for closing dropdowns, modals, or menus

AmwAutoFocusDirective

Automatically focuses an element when the component is initialized.

  • Selector: [amwAutoFocus]
  • Inputs: amwAutoFocus: boolean (default: true)
  • Usage: Apply to form inputs for immediate focus

AmwCopyToClipboardDirective

Copies text to the clipboard when an element is clicked.

  • Selector: [amwCopyToClipboard]
  • Inputs: amwCopyToClipboard: string
  • Outputs: (copied): EventEmitter<boolean>
  • Usage: Attach to buttons to enable copy functionality
